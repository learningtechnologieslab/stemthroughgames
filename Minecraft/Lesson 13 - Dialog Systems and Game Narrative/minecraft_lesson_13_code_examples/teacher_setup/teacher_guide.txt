---------------------------------------------------------------
RECOMMENDED WORLD SETUP (Teacher-Prepared)
---------------------------------------------------------------
Creating a starter world saves 10–15 minutes of setup time
and lets students focus on the dialogue/narrative design.

  1. Launch Minecraft Education Edition
  2. Create a new world:
       - Game mode: Creative
       - Difficulty: Peaceful
       - World name: "Day13_DialogueStarter"
  3. Build a small village:
       - 4–6 simple houses
       - A central square
       - A gate/entrance area
       - A tower for the extension NPC (Wizard Selene)
  4. Run the setup commands (copy-paste from chat):
       /scoreboard objectives add trust dummy
       /scoreboard objectives add questStarted dummy
       /scoreboard objectives setdisplay sidebar trust
  5. Place one NPC near the village centre
     (search "NPC" in Education Edition inventory)
  6. Export the world and share via your school's platform

  TIP: You do NOT have to pre-configure the NPCs. Leave them
  blank and let students set up the dialogue themselves —
  that is the core activity.

---------------------------------------------------------------
SHARING A WORLD WITH STUDENTS
---------------------------------------------------------------
  Option A — Direct share in the same session:
    Students join your world via "Join World" in the
    multiplayer menu. One teacher world, all students explore.

  Option B — Export and distribute:
    File → Export World → share the .mcworld file.
    Students import it via File → Import.
    Each student gets their own copy.

  Option C — Students build from scratch:
    Students create a new world themselves (Creative mode).
    They type setup commands themselves.
    Best for learning, takes a bit more time.

---------------------------------------------------------------
CLASSROOM COMMANDS — USEFUL SHORTCUTS
---------------------------------------------------------------
Run these from the teacher's game view to manage the class:

  Give all students operator status (lets them run /commands):
    /op @a

  Switch all students to Adventure Mode for testing:
    /gamemode adventure @a

  Switch all students back to Creative for building:
    /gamemode creative @a

  Teleport all students to you:
    /tp @a @s

  Reset all students' scoreboards at once:
    /scoreboard players set @a trust 0
    /scoreboard players set @a questStarted 0

  Show the trust scoreboard on everyone's screen:
    /scoreboard objectives setdisplay sidebar trust

  Clear the sidebar display:
    /scoreboard objectives setdisplay sidebar

---------------------------------------------------------------
COMMON MISTAKES AND HOW TO FIX THEM
---------------------------------------------------------------

  PROBLEM: Button doesn't do anything when clicked
  FIX: Check the "On-Click Command" field in the NPC editor.
       The command must start with / (forward slash).
       Also make sure the player is NOT in Creative Mode —
       NPC interactions behave differently in Creative.
       Switch to Adventure or Survival for testing.

  PROBLEM: /execute command gives an error
  FIX: Check for typos. The most common issues:
       • Missing "run" before the actual command:
           WRONG:  /execute if score @p trust matches 1 say hi
           RIGHT:  /execute if score @p trust matches 1 run say hi
       • Wrong selector: use @p for nearest player, @a for all
       • Wrong matches syntax: use matches 1 (not == 1 or = 1)

  PROBLEM: Scoreboard not created / "Unknown scoreboard" error
  FIX: Run /scoreboard objectives add trust dummy FIRST.
       This must be done before any command tries to read or
       write the "trust" variable.

  PROBLEM: Command Block not firing
  FIX: Check three things:
       1. Block type is correct (Impulse / Repeat / Chain)
       2. "Always Active" is enabled (not "Needs Redstone")
       3. For Chain blocks: they must be physically connected
          to the previous block and facing the right direction

  PROBLEM: Student gets the reward from Selene multiple times
  FIX: Make sure the rewardGiven scoreboard is being SET to 1
       after the first reward, and that the Repeat block
       checks  if score @s rewardGiven matches 0  as a condition.

  PROBLEM: Title/actionbar text isn't showing
  FIX: The JSON format for /title is strict. Check that:
       • Text is inside {"text":"..."} with double quotes
       • No curly-quote characters (" ") — use straight quotes
       • The command is on ONE line with no line breaks

---------------------------------------------------------------
EXIT TICKET — SUGGESTED ANSWERS
---------------------------------------------------------------

  Q1: What does "dialogue line 0" mean in your NPC story?
  A: The first line shown — position 0 is always the start
     of an ordered sequence, matching array index 0 in code.

  Q2: What happens if a player accesses a trust-locked branch
      without enough points?
  A: The /execute if score condition evaluates to false, so
     the command chain doesn't run. The player sees nothing
     (or the locked/default NPC dialogue).

  Q3: Name one way a Minecraft story differs from a book story.
  A: Any valid answer — e.g.: player choices branch the
     narrative; the world persists between sessions; the
     player's actions affect what happens next; variables
     (scoreboards) track player history.
